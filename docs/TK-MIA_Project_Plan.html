TK-MIA / NC Setup Sheet Project

Manufacturing Intelligence That Starts With Trustworthy NC Parsing

The goal is not to replace machinists, programmers, or engineers. The goal is to capture what the best people already know, organize it, and make every repeat job easier to set up, document, and improve.

First Priority: Make The NC Parser Reliable

No feature sprawl. The first milestone is a dependable NC parsing tool that can read real programs and produce structured data everyone can trust. Setup sheets, photo references, tool projections, inventory, and future integrations all depend on this foundation.

What We Are Building

TK-MIA is a local-first manufacturing assistant for NC program review, setup sheets, tool and fixture knowledge, photo documentation, and job-linked shop memory.

  • Reads NC programs and extracts tools, operations, offsets, feeds, speeds, and setup comments.
  • Creates setup sheets with practical ASCII visuals and photo placeholders.
  • Builds a repeatable knowledge base from real shop jobs.
  • Keeps every recommendation auditable and every original file untouched.

What Success Looks Like

A machinist opens a setup package and immediately understands what tools are needed, where the part zero is, what fixture is involved, what photos to check, and what changed since the last revision.

  • Less time searching for fixtures, taps, holders, and old setup notes.
  • Fewer unclear repeat jobs.
  • Faster first-piece setup.
  • Better training for newer machinists and better support for off-shifts.

Near-Term Data To Collect

  • Good NC programs from repeat jobs.
  • Bad or confusing NC programs for parser tests.
  • Setup sheets that already work well.
  • Tooling labels and holder notes.

Future Shop Rules

  • Minimum tool projections.
  • Tool serial numbers and asset IDs.
  • Tool, holder, jaw, and fixture storage locations.
  • Fixture naming standards.
  • Photo naming standards.
  • Approval and revision policies.

What We Avoid Early

  • Big database before parser proof.
  • Fancy graphics before clear data.
  • Automatic NC edits without approval.
  • Inventory sprawl before shop feedback.

The Human Part

The best version of TK-MIA is built with the people who actually run the jobs. Every photo, correction, label, setup note, and parser test makes the system more useful. This is how tribal knowledge becomes shared shop knowledge without taking respect away from the people who earned it.
